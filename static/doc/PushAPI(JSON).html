Introduction

 

Once the PUSH endpoints have been set up in the Anytrek Management System, the TrackerServer will automatically forward all the data packets from the devices. The protocol can be either HTTP or TCP, the contents are the all same. There will be two kinds of data packets: uPulse and uRpLoc, the first one for heartbeats and the second one for location reports.

Data Packets

uPulse

1.     Description

Heartbeat packet from devices.

2.     HTTP/TCP Content Example

{

"c":"uPulse",

"id":"86217603644154",

"psw":"000000",

"ver":32,

"t":"20180613140456",

"infoIndex":7,"settingIndex":1,

"sta":{"bat":3.982526,"wir":0,"charging":0,"acc":1},

"cid":{"system":"gsm","mcc":0,"mnc":0,"lac":0,"cid":0}

}

3.     HTTP/TCP Response (Optional)

{ErrorCode:#ErrorCode}

uRpLoc

1.     Description

Location report packet from devices.

2.     HTTP/TCP Content Example

{

"c":"uRpLoc",

"id":"86217603644154",

"psw":"000000",

"ver":32,

"t":"20180613183630",

"infoIndex":7, "settingIndex":1,

"gps":{"lng":"-117.557159","spd":0,"alt":0,"hdg":0,"lat":"34.063046"},

"sta":{"acc":1,"wir":43,"current":0,"bat":73,"charging":0},

"statistic":{"originalEvent":0,"event":0,"mileage":396}

}

3.     HTTP/TCP Response (Optional)

{ErrorCode:#ErrorCode}

ErrorCode

Error Code

Description

200

OK

400-01

invalid parameter or empty for required parameter

400-02

Device ID does not exist

400-03

Device ID is not activated

500-01

System Error
